Virtuoso ADE — Simulation Types Explained

DC Analysis
What it does: Finds DC operating point. Sweeps voltage/current sources. Current mirror use: Sweep I0 (1µA→100µA), plot IOUT vs IREF to verify mirroring ratio. Check Vgs, Vds, gm, ro of each transistor. Must run this FIRST before AC/tran.
DC Sweep
What it does: Parametric DC sweep. Current mirror use: Sweep VDC (Vout) from 0→1.8V with fixed I0. Plot IOUT to find output compliance voltage and current error vs ideal. Key metric: Vds_sat of M2.
AC Analysis
What it does: Small-signal frequency response. Requires DC operating point first. Current mirror use: Find output impedance Rout = IOUT/Vout over frequency. High Rout = better mirror. Check pole introduced by Cgs, Cgd of M2.
Transient
What it does: Time-domain response. Current mirror use: Apply a step/pulse to I0, observe IOUT settling time. Check for ringing or overshoot due to parasitic capacitances. Use stop time = 10× expected settling.
Parametric
What it does: Sweeps a design parameter across any analysis. Current mirror use: Sweep W/L ratio, temperature (−40 to 125°C), or VDD (1.6→2.0V). Essential to verify robustness before layout.
Noise
What it does: Computes output-referred and input-referred noise PSD. Current mirror use: Flicker noise (1/f) dominated by M1. Check at 1kHz and 1MHz.
Corner (ADE XL)
What it does: Runs simulation across process corners (TT, FF, SS, FS, SF) + temperature + voltage corners. Current mirror use: Verify IOUT accuracy across all 5 process corners. This is the real validation — not just TT.
Monte Carlo (ADE XL)
What it does: Statistical variation — process + mismatch. Current mirror use: Run 200+ iterations. Extract σ(IOUT/IREF). Mismatch on this W=50µm mirror should be <0.5% 3σ. If not, increase W or use cascode.

ADE Setup Sequence — Do This Every Time

Step 1
Launch → Virtuoso → ADE L (or ADE XL for corners). Setup → Model Libraries → add gpdk090 model file (.scs). Verify path is correct. This is the #1 cause of "model not found" errors.
Step 2
Setup → Simulator/Directory/Host → Select Spectre. Set simulation directory. Never leave it pointing to /tmp or a read-only path — causes file write errors.
Step 3
Variables → Copy From Cellview to import all schematic variables. Set I0 = 10µA. If variables are undefined, Spectre returns "dc operating point failed" — looks cryptic but the fix is simple.
Step 4
Analyses → Choose → dc. Run DC first and verify operating point: M1 and M2 should be in saturation (Vds > Vgs − Vth). Check in Results → Print → DC Operating Points.
Step 5
Simulation → Netlist and Run. If it fails — open the log file (simulation directory/spectre.out). The error is ALWAYS there. Never debug by guessing; read the log.

Most Common Virtuoso Errors + Exact Fix

Error 1
"ERROR (SPECTRE-16003): model not found" — Model library path wrong or not added. Fix: ADE → Setup → Model Libraries → check .scs file path for gpdk090.
Error 2
"DC operating point failed to converge" — Usually bad initial conditions or floating node. Fix: Check all nodes have a DC path to GND. Check I0 source is not too large for device sizes.
Error 3
"undefined variable" — Variables not set in ADE. Fix: ADE → Variables → Copy From Cellview, then set all values explicitly. Never leave variables blank.
Error 4
"port not found" or "net unconnected" — Check pin on symbol doesn't match schematic. Verify gate of M2 is connected to gate+drain of M1. Blue dot = connected in Virtuoso.
Error 5
"cannot write to simulation directory" — Permissions issue. Fix: Set simulation dir under your home directory. ADE → Setup → Simulator/Directory → change path to ~/simulation/cellname.
Error 6
AC analysis returns garbage / flat line — AC analysis requires a DC op point AND an AC stimulus. Check: VDC source needs ac=1 set, or add a separate AC current source iprobe at the output.
Error 7
LVS "short" or "open" errors after layout — Most common: missing via, unconnected metal layer, or body tie missing on NMOS bulk. In gpdk090 NMOS: bulk must connect to GND explicitly in layout.
